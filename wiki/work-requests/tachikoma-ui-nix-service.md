# Tachikoma UI — Nix Service Declaration

Declare the Tachikoma UI as a launchd service in personal-nix so it auto-starts on login and is always available at http://localhost:4000.

## Goal

The Tachikoma UI Express server (`mcps/tachikoma-ui/server/index.ts`) should start automatically on login as a launchd user agent, managed by nix-darwin + home-manager. It reads `ANTHROPIC_API_KEY` from `~/.secrets`. The frontend must be built (`npm run build`) before the server starts so it can serve `dist/` as static files.

## Context

### What already exists

- `mcps/tachikoma-ui/` — complete app (server + frontend, built by the prior work request)
- `default.nix` — personal home-manager module, imports `packages.nix` and `mcp.nix`
- `mcp.nix` — MCP server registration via `home.activation.registerMCPServers`
- `mcps/tachikoma-mcp/` — example of an adjacent Node.js service (not a launchd service — MCPs are on-demand)

### How launchd services work in this repo

Services are declared via `launchd.user.agents` in home-manager. This is already used by the team `dev-environment` for Docker Desktop, AltTab, etc. The same mechanism is available in `personal-nix/default.nix` since it's a home-manager module.

Pattern:
```nix
launchd.user.agents.my-service = {
  serviceConfig = {
    ProgramArguments = [ "/path/to/node" "--flag" "/path/to/script.ts" ];
    RunAtLoad = true;
    KeepAlive = true;
    StandardOutPath = "/tmp/my-service.log";
    StandardErrorPath = "/tmp/my-service.log";
    EnvironmentVariables = {
      KEY = "value";
    };
  };
};
```

Node.js binary path from nixpkgs: `${pkgs.nodejs_22}/bin/node`

### Activation sequence needed

Before the launchd service can serve the frontend, `npm install` and `npm run build` must run in `mcps/tachikoma-ui/`. Add a `home.activation` step that runs these after `writeBoundary`. This ensures deps and the `dist/` folder are fresh after every `dev` rebuild.

### Secret: ANTHROPIC_API_KEY

`~/.secrets` is generated by the `secretsFromKeychain` activation step (already in `default.nix`). The launchd service needs to read the key at agent start time, not at nix build time (since the value is dynamic). 

Best approach: write a small wrapper script that sources `~/.secrets` and then execs the node server. Declare the wrapper in the activation script or as a nix-generated file. Store the wrapper at `~/.local/bin/tachikoma-ui-start` (or similar).

Alternative simpler approach: use `EnvironmentVariables` in the launchd plist with a placeholder, and override via a `launchd.user.agents.tachikoma-ui.serviceConfig.EnvironmentVariables.ANTHROPIC_API_KEY` that is set dynamically. 

The cleanest approach that matches existing patterns in this repo: write the wrapper script from the activation step, after `secretsFromKeychain` runs and sources `~/.secrets`. The wrapper script reads the key from `~/.secrets` at runtime (not build time), so it stays fresh even if the key rotates.

## Implementation plan

1. Add a `home.activation.buildTachikomaUI` step in `default.nix` (after `writeBoundary`) that:
   - Runs `npm install --quiet` in `$HOME/projects/personal-nix/mcps/tachikoma-ui`
   - Runs `npm run build` in the same directory
   - Non-fatal on failure (prints warning, doesn't crash rebuild)

2. Add a `home.activation.writeTachikomaUILauncher` step (after `secretsFromKeychain` and `buildTachikomaUI`) that writes `~/.local/bin/tachikoma-ui-start`:
   ```bash
   #!/bin/bash
   set -a
   [ -f "$HOME/.secrets" ] && . "$HOME/.secrets"
   set +a
   exec /path/to/node --experimental-strip-types \
     "$HOME/projects/personal-nix/mcps/tachikoma-ui/server/index.ts"
   ```
   Make it executable with `chmod +x`.

3. Declare `launchd.user.agents.tachikoma-ui` in `default.nix`:
   ```nix
   launchd.user.agents.tachikoma-ui = {
     serviceConfig = {
       ProgramArguments = [ "${config.home.homeDirectory}/.local/bin/tachikoma-ui-start" ];
       RunAtLoad = true;
       KeepAlive = true;
       StandardOutPath = "/tmp/tachikoma-ui.log";
       StandardErrorPath = "/tmp/tachikoma-ui.log";
     };
   };
   ```

## Stop condition

1. `default.nix` has a `home.activation.buildTachikomaUI` step that runs `npm install` and `npm run build` in `mcps/tachikoma-ui/`
2. `default.nix` has a `home.activation.writeTachikomaUILauncher` step that writes and chmod-execs `~/.local/bin/tachikoma-ui-start`
3. The launcher script sources `~/.secrets` and execs `node --experimental-strip-types .../server/index.ts`
4. `default.nix` declares `launchd.user.agents.tachikoma-ui` with `RunAtLoad = true`, `KeepAlive = true`, and log paths
5. `nix eval --impure .#homeConfigurations` (or equivalent syntax check) exits without errors — the nix expressions are valid
6. No secrets are hardcoded in any nix file — the key is read at runtime from `~/.secrets` via the launcher script
