## Iter 14 — SCCACHE_DIR fix for cc-crate/sccache HOME conflict
- Root cause found: the cc crate (used by ring, whisper-rs-sys) automatically wraps C compilation through sccache when RUSTC_WRAPPER=sccache is set. In nix build environments, HOME=/homeless-shelter (read-only), so sccache fails to create its preprocessor cache dir → exit 254 on ring's curve25519.c.
- Fix: added `SCCACHE_DIR = "${homeDir}/Library/Caches/sccache"` to all three crane derivations (cargoArtifacts, proxy-daemon-pkg, proxy-voice-pkg) so sccache uses the real host cache path regardless of $HOME.
- Also added pname/version to all crane derivations to silence evaluation warnings (virtual workspace root has no [package] section).
- Committed to worktree, rebased onto origin/master, pushed, opened PR #22.
- Applied fix directly to live ~/projects/personal-nix/modules/proxy-rust-services.nix so user can run `dev` immediately without waiting for merge.

## Iter 15 — root cause: sccache SERVER cannot spawn compiler in nixbld build dirs
- Diagnosed why bhefmuquv (no SCCACHE_NO_DAEMON) failed: "sccache: encountered fatal error: failed to spawn Command { cd /nix/var/nix/builds/... }" — the sccache server (pioneer) cannot cd into nixbld's build directory (drwx------ mode 700). When the sccache server is running, it receives compile requests and tries to spawn the compiler in the build dir; pioneer lacks permission.
- Root cause chain: b89yfufwb succeeded because the sccache server was NOT running at that time (sccache fell back to inline/local-daemon mode running as nixbld). bhefmuquv failed because server was running.
- Final fix: SCCACHE_NO_DAEMON=1 forces inline mode (sccache runs as the nixbld user, which owns its build dir). sccache server is bypassed entirely. Fixed comment to explain the server spawn failure clearly.
- /private/tmp/nix-sccache is owned by _nixbld1 (mode 750). Sequential builds (cargoArtifacts → proxy-daemon → proxy-voice) are likely assigned to _nixbld1, so _nixbld1 owns the dir and can write cache entries. Cross-run cache hits should work.
